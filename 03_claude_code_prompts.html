DOC 03 — Claude Code Prompts

Prompts Exactos
Claude Code

Prompts listos para pegar en Claude Code (terminal). Cubren el setup del backend del Panel de Creadores con Supabase Auth, Instagram Graph API, y Meta Business API.

00 — Contexto del proyecto

Qué existe y qué falta.

El Panel de Creadores está vivo en producción en allver-social-dashboard.vercel.app, construido en Next.js y deployado en Vercel. Tiene 6 secciones: /instagram, /analytics, /calendar, /competitors, /news, /agents. El repositorio está en GitHub como "allver social dashboard".

Lo que falta es todo el backend: autenticación real, conexión con la Instagram Graph API para obtener métricas reales, conexión con Meta Business API para datos de ads, base de datos de leads, y un sistema de recursos para los creadores. El objetivo de esta semana es tener una versión usable que Mauro pueda usar.

02 — Prompt de Supabase Auth

Implementar autenticación completa.

Una vez que Claude Code hizo el diagnóstico inicial del repo, usa este prompt para arrancar con el auth. Este es el paso más crítico — sin auth, nada más tiene sentido.

# DESPUÉS DEL DIAGNÓSTICO — IMPLEMENTAR AUTH

Implementa autenticación completa con Supabase en este proyecto Next.js.

PASOS EN ORDEN:

1. Instala las dependencias necesarias:
   @supabase/supabase-js
   @supabase/auth-helpers-nextjs
   @supabase/auth-ui-react (para el componente de login listo)

2. Crea el cliente de Supabase en lib/supabase.ts

3. Crea las variables de entorno necesarias:
   NEXT_PUBLIC_SUPABASE_URL
   NEXT_PUBLIC_SUPABASE_ANON_KEY
   (Dime exactamente dónde conseguirlas en el dashboard de Supabase)

4. Crea la página /login con:
   - Formulario email + password
   - Botón "Continuar con Google"
   - Design consistente con el panel existente (oscuro, minimal)

5. Crea el middleware de protección de rutas:
   - Todas las rutas excepto /login requieren auth
   - Sin sesión activa → redirect automático a /login
   - Con sesión → acceso normal al panel

6. Crea el hook useUser() para acceder al usuario en cualquier componente

7. Agrega el botón de logout en el header/navbar del panel

IMPORTANTE:
- Usa el App Router de Next.js (no Pages Router)
- Server Components donde sea posible para el auth
- El middleware va en middleware.ts en la raíz del proyecto
- Dime exactamente qué tablas necesito crear en Supabase manualmente

Después de implementar, verifica que funciona correctamente y dame 
las instrucciones exactas para configurar Supabase (qué hacer en el dashboard).
03 — Prompt de Instagram Graph API

Conectar datos reales de Instagram.

Este es el prompt que convierte el panel de un mockup estático en una herramienta real. Se pega después de que el auth esté funcionando.

# DESPUÉS DEL AUTH — INSTAGRAM GRAPH API

Implementa la conexión con Instagram Graph API para obtener métricas reales.

CONTEXTO:
Los entrenadores tienen cuentas de Instagram Business o Creator.
Necesitamos conectar su cuenta al panel y mostrar sus métricas reales.

FLUJO DE OAUTH DE INSTAGRAM:
1. El creador hace click en "Conectar Instagram" en su panel
2. Se abre el flujo de OAuth de Meta/Instagram
3. El creador autoriza los permisos necesarios
4. Guardamos el access_token en Supabase (tabla creators)
5. Usamos el token para llamar a la API en tiempo real

PERMISOS NECESARIOS (solicitar en el OAuth):
- instagram_basic
- instagram_content_publish
- instagram_manage_insights
- pages_read_engagement

DATOS A OBTENER Y MOSTRAR EN /instagram:
- Followers actuales y delta últimos 30 días
- Reach total últimos 30 días
- Impresiones últimos 30 días
- Engagement rate promedio
- Últimos 12 posts con: imagen thumbnail, likes, comments, reach, engagement individual
- Mejores horas de publicación (si la API lo provee)

ENDPOINTS DE LA API A USAR:
- GET /{ig-user-id}?fields=followers_count,media_count
- GET /{ig-user-id}/insights?metric=reach,impressions,follower_count&period=month
- GET /{ig-user-id}/media?fields=id,caption,media_type,thumbnail_url,timestamp,like_count,comments_count

IMPLEMENTACIÓN:
1. Crea /app/api/instagram/connect/route.ts → maneja el OAuth callback
2. Crea /app/api/instagram/metrics/route.ts → llama a la Graph API y retorna datos
3. Guarda el token encriptado en Supabase (columna instagram_access_token en tabla creators)
4. En /instagram, detecta si el creador tiene token:
   - Sin token: muestra botón "Conectar tu Instagram"
   - Con token: muestra las métricas reales en tiempo real

IMPORTANTE:
- La Graph API requiere una Meta App configurada. 
  Dame el paso a paso exacto de cómo crear la app en developers.facebook.com
- Los tokens de Instagram expiran — implementa refresh automático
- Maneja errores de API gracefully (mostrar mensaje amable si falla)
04 — Prompt del Sistema de Contenido

El calendario de contenido inteligente.

Este prompt construye la sección más diferenciadora del panel: el sistema que le dice al entrenador exactamente qué publicar y cuándo, basado en su avatar y oferta previamente definidos.

# SISTEMA DE CONTENIDO — SECCIÓN /calendar

Construye el sistema de calendario de contenido inteligente para el panel.

CONTEXTO DEL CREADOR (Mauro):
- Entrenador presencial, Medellín, Colombia
- Niche: fitness de alto rendimiento para hombres ejecutivos
- Oferta: programa de transformación física para recuperar energía y autoridad
- Objetivo: 5-10 DMs calificados por día desde Instagram
- Meta: primeras ventas de $1,500 USD en los próximos 30 días

SECCIÓN /calendar — LO QUE NECESITA MOSTRAR:

1. VISTA DE CALENDARIO MENSUAL
   - 30 días con contenido pre-cargado
   - Cada día muestra: tipo de post, hora sugerida de publicación, status (pendiente/publicado)
   - Color coding: Reel (azul), Carrusel (verde), Story (naranja), DM outreach (morado)

2. DETALLE DE CADA POST (al hacer click en un día)
   - Tipo: Reel / Carrusel / Story
   - Hook principal (el primer texto que se ve)
   - Caption completo listo para copiar
   - Hashtags sugeridos
   - Hora óptima de publicación
   - Objetivo del post: atraer / calificar / generar DMs

3. BASE DE DATOS NECESARIA (tabla content_calendar):
   creator_id, fecha, tipo_post (reel/carousel/story/dm),
   hook, caption, hashtags, hora_publicacion, status, objetivo

4. PRECARGA LOS PRIMEROS 30 DÍAS DE CONTENIDO PARA MAURO
   Siguiendo el Sistema Carrizo:
   - Semana 1: Reels de identidad y visión (quién es Mauro, su historia)
   - Semana 2: Carruseles de autoridad (resultados de clientes, método)
   - Semana 3: Mix de Reels + Stories de conversación (generar DMs)
   - Semana 4: Contenido de cierre (llamadas a la acción directas)

5. GENERADOR DE HOOKS CON IA (bonus si tienes tiempo)
   - Botón "Generar nuevo hook" que llame a la API de Claude
   - Input: tipo de post + objetivo del día
   - Output: 3 opciones de hook para elegir

UI DEL CALENDARIO:
- Grid mensual con días clickeables
- Panel lateral que abre el detalle del post al hacer click
- Botón "Marcar como publicado" para tracking
- Export a Google Calendar (opcional, si el tiempo alcanza)

Asegurar que todo funciona en producción.

Antes de mostrarle a Mauro, usa este prompt para que Claude Code haga una revisión completa y el deploy final.

Antes de hacer el deploy final, necesito una revisión completa del proyecto.

CHECKLIST DE VERIFICACIÓN:

AUTH:

□ Rutas protegidas redirigen a /login si no hay sesión

06 — Orden de ejecución recomendado

La secuencia exacta para esta semana.

01

Abrir el repo en Claude Code Terminal

Navegar a la carpeta del proyecto en el local. Verificar que tiene el repositorio correcto. Pegar el Prompt 01 (Setup Inicial) para que Claude Code haga el diagnóstico completo del repo antes de tocar nada.

02

Crear proyecto en Supabase Dashboard

Ir a supabase.com → New Project. Guardar las credenciales (URL y anon key). Crear las tablas que Claude Code te indique. Configurar OAuth de Google en Authentication → Providers.

03

Implementar Auth Claude Code

Pegar el Prompt 02. Dejar que Claude Code escriba todo el código. Solo intervenir si pide las credenciales o hay una decisión de diseño. El auth completo no debería tomar más de 30–40 minutos de sesión.

04

Crear Meta App developers.facebook.com

Ir a developers.facebook.com → Crear App de tipo Business. Agregar el producto Instagram Graph API. Configurar los redirect URIs con la URL de producción. Guardar App ID y App Secret.

05

Implementar Instagram API Claude Code

Pegar el Prompt 03. Este es el paso más complejo. Puede requerir iteración. Si Claude Code se traba en algún error de la API, pídele que explique el problema y busca la solución en la documentación oficial de Meta.

06

Sistema de Contenido de Mauro Claude Code

Pegar el Prompt 04. Aquí es donde el panel se vuelve realmente valioso para Mauro. Los 30 días de contenido precargados son el wow-factor que lo convencerá de que el sistema funciona.

07

Deploy y verificación final Claude Code

Pegar el Prompt 05. Verificar que todo funciona en producción. Antes de mostrarle a Mauro, hacer el recorrido completo del usuario como si fueras él: login, ver el calendario, ver el estado de Instagram, marcar un post como publicado.

Nota importante sobre tiempo estimado El auth + Instagram API + calendario son realistas para 2–3 sesiones intensas de Claude Code de 2–3 horas cada una. No lo fuerces todo en una noche — especialmente la Instagram API que puede tener configuraciones del lado de Meta que requieren tiempo de aprobación.
