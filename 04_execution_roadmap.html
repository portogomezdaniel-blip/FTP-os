00 — Norte de la semana

Una semana, dos victorias concretas.

Al finalizar esta semana, el resultado tiene que ser exactamente esto y nada más: el Panel de Creadores con auth + Instagram API funcionando en producción, y Mauro con su primer mes de calendario de contenido cargado y listo para ejecutar. Todo lo demás es secundario.

La tentación va a ser querer construir 5 cosas al mismo tiempo porque todas son interesantes. La disciplina esta semana es terminar antes de empezar. Mauro con el panel funcionando es el primer caso de éxito del Digital Operator Pack — ese caso de éxito vale más que cualquier otra feature nueva.

Victoria 1 — Panel de Creadores usable Auth funcionando, Instagram API conectada, calendario de 30 días de Mauro cargado. Deployed en Vercel. Mauro puede entrar, ver sus métricas, y seguir su plan de contenido.
Victoria 2 — Mauro activado Mauro está publicando con el sistema. Tiene su hook del primer Reel. Sabe exactamente qué publicar los próximos 30 días. Andrea Engine está listo para manejar sus DMs.

02 — Matriz de prioridades

Qué hacer, qué delegar, qué ignorar.

🔴 Urgente + Importante

Auth del panel. Instagram API. Primer post de Mauro. Deploy estable. Estas cosas bloquean todo lo demás si no se resuelven.

🟡 Importante pero no urgente

Sistema de recursos del panel. Email sequences. Lead magnet de Mauro. Food Trucks backend. Buscar nuevos clientes para el Digital Operator Pack.

🔵 Urgente pero delegable

Producción visual de los carruseles de Mauro (Canva). Transcripción y edición de Reels. Respuestas a DMs básicos (Andrea Engine).

⚫ Ni urgente ni importante

Nuevas features del panel que Mauro no necesita todavía. Investigar herramientas nuevas. Consumir más contenido sobre sistemas de infoproductores.

03 — Trampas a evitar esta semana

Los patrones que matan el momentum.

Trampa 1 — Empezar una tercera cosa antes de terminar las dos Esta semana son dos victorias: panel funcionando + Mauro activado. Si aparece una idea nueva (suplementos, nuevo cliente, nueva feature), se anota y va para la semana que viene. No esta semana.
Trampa 2 — Perfeccionar el panel antes de mostrárselo a Mauro El panel no tiene que estar perfecto para mostrárselo. Tiene que funcionar. El feedback de Mauro en vivo vale más que 3 días más de desarrollo en solitario.
Trampa 3 — Quedarse pegado en un error de la API de Meta La API de Instagram/Meta puede ser frustrante. Si Claude Code lleva más de 45 minutos en el mismo problema, escalar el error aquí en Claude AI para diagnóstico antes de seguir. No pelear solo con la documentación de Meta.
Trampa 4 — Construir para el cliente hipotético en lugar de para Mauro Cada feature del panel esta semana se valida con una pregunta: ¿Mauro necesita esto esta semana? Si la respuesta es no, va para después. El over-engineering es la forma más elegante de evitar mostrar el producto.
04 — Índice de documentos del kit

Los 5 archivos de esta sesión.

Todos los documentos de esta sesión están en el mismo directorio. Si vas a abrir una nueva sesión en cualquier terminal, la secuencia es: leer primero el 00_master_brief para reconstruir el contexto completo, luego el documento específico del proyecto en el que vas a trabajar.

El archivo 00_master_brief.html es el mapa del holding completo — el punto de entrada para cualquier sesión nueva. El 01_carrizo_blueprint.html tiene todos los datos del sistema Carrizo para usarlos en el trabajo con Mauro o en la venta del Digital Operator Pack. El 02_digital_operator_pack.html es el producto estructurado con precios, fases y entregables. El 03_claude_code_prompts.html tiene los prompts exactos para pegar en Claude Code en orden de ejecución. Y este documento, el 04_execution_roadmap.html, es el plan de acción día a día.
